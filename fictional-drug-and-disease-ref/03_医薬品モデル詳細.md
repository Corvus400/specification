## 医薬品データモデル

### 分類軸

| 軸 | 値の数 | 画面分岐への影響 | 出典 |
|----|-------|------------------|------|
| 投与経路 (routeOfAdministration) | 8 (`RouteOfAdministration` enum) | 用法用量 UI の表現形式、適用上の注意の要否 | [KEGG BRITE ATC分類](https://www.kegg.jp/brite/jp08303) |
| ATC 分類 (atcCode) | 第 1 階層 14 群 (`A` `B` `C` `D` `G` `H` `J` `L` `M` `N` `P` `R` `S` `V`) | 薬効カテゴリ絞り込み検索、関連薬品表示 | 同上 |
| 規制区分 (regulatoryClass) | 11 (`RegulatoryClass` enum、複数値可) | 詳細画面先頭バッジ、警告色強調 | [医薬品添付文書レイアウト](https://www.data-index.co.jp/knowledge/141/) |
| 剤形 (dosageForm) | 13 (`DosageForm` enum) | 組成・性状の表示構造、画像生成分岐 | 同上 |

### 共通フィールド (全医薬品で必須)

添付文書 [新記載要領 26 項目 (令和5年改正)](https://www.mhlw.go.jp/content/11120000/001060028.pdf) に準拠。架空データでは以下を必須採用。型が入れ子 data class / enum 参照の場合は下記「入れ子構造」「列挙型定義」に展開。

| # | フィールド | 型 | 必須 | 添付文書対応項目 | 備考 |
|---|------------|-----|------|------------------|------|
| 1 | id | `String` | 必須 | — | `drug_NNNN` 4 桁ゼロ埋め (例: `drug_0001`) |

| 3 | brandName | `String` | 必須 | 名称 (販売名) | 架空名 |

| 6 | yjCode | `String?` | 任意 | 日本標準商品分類番号 | 架空の 12 桁コード (例: `"999171158264"`) |

### 入れ子構造 (全て `@Serializable data class`)

#### NumberedParagraph (番号付き段落 — 汎用)

番号付き段落を持つ全フィールド (warning / contraindications / importantPrecautions 等) で共通使用。

```kotlin
@Serializable
data class NumberedParagraph(
    val order: Int,              // 表示順 (1, 2, 3 ...)
    val subOrder: Int? = null,   // サブ番号 (1.1 の 2 階層目)。null ならフラット
    val content: String,         // 記法: Markdown (短文なら平文、長文なら段落 Markdown)
)
```

表示規約 (クライアント側レンダリング):

- フィールドごとに添付文書項番が確定しているため、`order` / `subOrder` はフィールド内ローカルな連番として扱う
- 添付文書慣例の表示フォーマット:
  - `warning` フィールドの要素: "1.1." "1.2." (親項番 "1" は警告固定) → クライアントは `"1.${order}."` でレンダリング
  - `contraindications` フィールドの要素: "2.1." "2.2." → `"2.${order}."`
  - `importantPrecautions` フィールドの要素: "8.1." "8.2." → `"8.${order}."`
  - 以下同様 (フィールド名から項番を決定)
- `subOrder` が非 null のときは `"{親項番}.{order}.{subOrder}."` (例: "8.1.2.")

#### InteractionInfo (相互作用)

```kotlin
@Serializable
data class InteractionInfo(

@Serializable
data class InteractionEntry(
```

#### AdverseReactionInfo (副作用)

```kotlin
@Serializable
data class AdverseReactionInfo(
    val serious: List<AdverseReaction> = emptyList(),    // 重大な副作用 (独立項、UI 警告色強調)
    val other: AdverseReactionByFrequency,               // その他の副作用 (頻度別)
)

@Serializable
data class AdverseReaction(
    val name: String,
    val frequency: FrequencyBand,    // 頻度帯 enum
    val symptom: String,             // 記法: Markdown (1-2 文)
    val initialSigns: String,        // 初期症状 (平文)
    val countermeasure: String,      // 対応 (平文 1-2 文)
)

@Serializable
data class AdverseReactionByFrequency(
    val over5Percent: List<String> = emptyList(),           // 頻度 5% 以上 → JSON: "over5_percent"
    val between1And5Percent: List<String> = emptyList(),    // 頻度 1-5% → JSON: "between1_and5_percent"
    val under1Percent: List<String> = emptyList(),          // 頻度 1% 未満 → JSON: "under1_percent"
    val frequencyUnknown: List<String> = emptyList(),       // 頻度不明 → JSON: "frequency_unknown"
)
```

`AdverseReactionByFrequency` の JSON キーは `JsonNamingStrategy.SnakeCase` のフィールド名変換結果として `over5_percent` / `between1_and5_percent` / `under1_percent` / `frequency_unknown` になる (フィールド名内の数字位置で区切り位置が決まり、enum `FrequencyBand` の SerialName `over_5_percent` 等とは別物)。各要素は短い副作用名 (1-10 字、MD 非対象)。

### 列挙型定義

全て `@Serializable enum class`、JSON ワイヤ値は `@SerialName` で **英語 snake_case** に固定する。Kotlin source / IDE hover 上の人間可読性は KDoc で日本語ラベルを保持する。

```kotlin
@Serializable
enum class RegulatoryClass {
    /** 毒薬 */ @SerialName("poison") POISON,
    /** 劇薬 */ @SerialName("potent") POTENT,
    /** 普通薬 */ @SerialName("ordinary") ORDINARY,
    /** 向精神薬第1種 */ @SerialName("psychotropic_1") PSYCHOTROPIC_1,
    /** 向精神薬第2種 */ @SerialName("psychotropic_2") PSYCHOTROPIC_2,
    /** 向精神薬第3種 */ @SerialName("psychotropic_3") PSYCHOTROPIC_3,
    /** 麻薬 */ @SerialName("narcotic") NARCOTIC,
    /** 覚醒剤原料 */ @SerialName("stimulant_precursor") STIMULANT_PRECURSOR,
    /** 生物由来製品 */ @SerialName("biological") BIOLOGICAL,
    /** 特定生物由来製品 */ @SerialName("specified_biological") SPECIFIED_BIOLOGICAL,
    /** 処方箋医薬品 */ @SerialName("prescription_required") PRESCRIPTION_REQUIRED,
}

@Serializable
enum class DosageForm {
    /** 錠剤 */ @SerialName("tablet") TABLET,
    /** カプセル */ @SerialName("capsule") CAPSULE,
    /** 散剤 */ @SerialName("powder") POWDER,
    /** 顆粒 */ @SerialName("granule") GRANULE,
    /** 液剤 */ @SerialName("liquid") LIQUID,
    /** 注射剤 */ @SerialName("injection_form") INJECTION_FORM,
    /** 軟膏 */ @SerialName("ointment") OINTMENT,
    /** クリーム */ @SerialName("cream") CREAM,
    /** 貼付剤 */ @SerialName("patch") PATCH,
    /** 点眼液 */ @SerialName("eye_drops") EYE_DROPS,
    /** 坐剤 */ @SerialName("suppository") SUPPOSITORY,
    /** 吸入剤 */ @SerialName("inhaler") INHALER,
    /** 点鼻液 */ @SerialName("nasal_spray") NASAL_SPRAY,
}

@Serializable
enum class RouteOfAdministration {
    /** 内服 */ @SerialName("oral") ORAL,
    /** 外用 */ @SerialName("topical") TOPICAL,
    /** 注射 */ @SerialName("injection_route") INJECTION_ROUTE,
    /** 吸入 */ @SerialName("inhalation") INHALATION,
    /** 直腸内投与 */ @SerialName("rectal") RECTAL,
    /** 点眼 */ @SerialName("ophthalmic") OPHTHALMIC,
    /** 点鼻 */ @SerialName("nasal") NASAL,
    /** 経皮 */ @SerialName("transdermal") TRANSDERMAL,

@Serializable
enum class PrecautionPopulationCategory {

    /** 肝機能障害 */ @SerialName("hepatic_impairment") HEPATIC_IMPAIRMENT,
    /** 生殖能有する患者 */ @SerialName("reproductive_potential") REPRODUCTIVE_POTENTIAL,
    /** 妊婦 */ @SerialName("pregnant") PREGNANT,
    /** 授乳婦 */ @SerialName("lactating") LACTATING,
    /** 小児等 */ @SerialName("pediatric") PEDIATRIC,
    /** 高齢者 */ @SerialName("geriatric") GERIATRIC,
}

@Serializable
enum class HepaticSeverity {
    /** 軽度 */ @SerialName("mild") MILD,
    /** 中等度 */ @SerialName("moderate") MODERATE,
    /** 重度 */ @SerialName("severe") SEVERE,

@Serializable
enum class RenalSeverity {
    /** 正常 (CrCl ≥ 90) */ @SerialName("normal") NORMAL,
    /** 軽度低下 (60-89) */ @SerialName("mild") MILD,
    /** 中等度低下 (30-59) */ @SerialName("moderate") MODERATE,
    /** 重度低下 (15-29) */ @SerialName("severe") SEVERE,

    /** 1% 未満 */ @SerialName("under_1_percent") UNDER_1_PERCENT,

    /** μg */ @SerialName("microgram") MICROGRAM,

    /** L */ @SerialName("l") L,

@Serializable
enum class StorageTemperature {
    /** 室温 */ @SerialName("room_temperature") ROOM_TEMPERATURE,
    /** 冷所 (2-8℃) */ @SerialName("cold") COLD,
```

<a id="combination-counts"></a>

## 組合せ数計算 (医薬品)

### 計算前提

「データ件数」= 1 Drug レコード単位 (対応する疾患側は [04_疾患モデル詳細.md `#combination-counts`](04_疾患モデル詳細.md#combination-counts) 参照)。UI 発火条件 (非空 / 非 null / 特定 enum 値) を全網羅しつつ、分類軸 × UI ブロックの組合せを最小件数で満たすカバレッジを求める。素案は「薬品 50 件」だが、以下の計算で**実現不可能**であることを示す。

### (A) 分類軸の全値カバレッジ

| 軸 | 値の数 | 必要件数 (単独) |
|----|-------|----------------|
| ATC 第 1 階層 | 14 群 | 14 |
| DosageForm | 13 種 | 13 |
| RouteOfAdministration | 8 種 | 8 |
| RegulatoryClass | 11 種 | 11 |

**A の最大値**: `max(14, 13, 8, 11) = 14` (全軸 1 件で兼ねる理論下限)。
ただし「外用軟膏 × A 群 (消化器代謝)」のような実在しない組合せがあり、軸間の独立性は保証されない。ペアワイズカバレッジで考えると次項。

### (B) ペアワイズ・カテゴリ組合せ

ATC 14 × DosageForm 13 = 182 論理ペア。実在可能ペアは約 70 (例: J 群は注射剤・錠剤中心、D 群は軟膏・クリーム中心)。ペアワイズテスティング (直交表 L_n) で圧縮すると **20-25 件**で全有効ペアを網羅できる。

### (C) UI ブロック発火条件

| ブロック | 発火に必要なレコード数 |
|---------|---------------------|
| D3 warning 非空 | 1 |
| D8 用法詳細タブ (age × renal × hepatic の 2^3 組合せ) | 8 |
| D9 importantPrecautions 非空 | 1 |
| D10 precautionsForSpecificPopulations 8 カテゴリ網羅 | 8 |
| D11 interactions 4 状態 (null / 禁忌のみ / 注意のみ / 両方) | 4 |
| D12 serious 非空 | 1 |
| D13 AdverseReactionByFrequency 4 頻度帯網羅 | 4 |
| D14 pharmacokinetics 非 null | 1 |
| D15 補助情報 6 フィールド各非空 | 6 |
| D16 handlingPrecautions / insuranceNotes 各非空 | 2 |
| D17 approvalConditions / references 各非空 | 2 |
| D18 relatedDiseaseIds 非空 | 1 |

**C の単純加算**: 38 (ただし 1 件で複数ブロックを兼ねる重複圧縮可)。

### (D) 分類軸別追加フィールド発火

| 分類 | 必要件数 |
|------|---------|
| 注射剤 (静脈 / 筋肉 / 皮下 の用途別) | 3 |
| 外用剤 6 種 (軟膏 / クリーム / 貼付 / 点眼 / 点鼻 / 坐剤) | 6 |
| 生物由来製品 (普通 / 特定 の 2 区分) | 2 |
| 毒薬 | 2 |
| 劇薬 | 2 |
| 向精神薬 3 種 | 3 |
| 麻薬 | 1 |
| 覚醒剤原料 | 1 |
| 慢性疾患長期服用薬 (`atcCode` 第 1 文字 `A` or `C` 全般) | 3 |

**D の単純加算**: 23 (A, B, C と重複可)。

### (E) 重複圧縮後の最小値

単純加算 = A(14) + C(38) + D(23) + ペアワイズ余剰 = 約 75。
1 件で平均 2 条件を兼ねるとすると 約 38 件で下限に達するが、UI ブロック 8 (用法詳細 2^3) と D10 (8 カテゴリ) は厳格に 8 件ずつ必要で兼用不可 → 下限は**約 55-60 件**。

余剰 (テスト可用性・バリエーション・ATC 主要群への集中分布) を考慮し、

**医薬品推奨件数: 120 件**
