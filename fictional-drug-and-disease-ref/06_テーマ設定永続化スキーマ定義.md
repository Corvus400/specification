# テーマ設定永続化スキーマ定義

親プラン: [05_医薬品・疾患マスタアプリ詳細化プラン.md][plan] (永続化設計)

## 目的

ユーザーのテーマ選択 (システム追従 / ライト / ダーク) を永続化する。本アプリの UI はシステムテーマに自動追従し、テーマ切替 UI は提供しない。永続化キーは将来のユーザー切替 UI 追加に備えてキー名と値域を定義する。

## 永続化形式

key-value ストア形式 (Flutter `shared_preferences` / iOS `@AppStorage`)。テーブル DDL は不要。

### 永続化キー

| キー名 | 型 | デフォルト値 | 値域 | 説明 |
|-------|----|------------|-----|------|
| `theme_mode` | String | `system` | `system` / `light` / `dark` | テーマモード |

アクセス方法:
- Flutter: `SharedPreferences.getString("theme_mode")` / `setString("theme_mode", value)`
- iOS: `@AppStorage("theme_mode") var themeMode: String = "system"`

## 使用箇所

本アプリは UI からのテーマ切替手段を提供せず、システムテーマに自動追従する。

- 表示: Flutter `MediaQuery.platformBrightness` / iOS `@Environment(\.colorScheme)` でシステム値を読み取り、`MaterialApp.themeMode` / SwiftUI のカラースキームに適用
- `theme_mode` キーへの読み書きは行わない (キー予約のみ)

ユーザー切替 UI を追加する場合は別プランで設計し、本ファイルの値域定義に従って永続化する。

## ライフサイクル

| 操作 | トリガ | 挙動 |
|-----|-------|------|
| 読み取り | — | 行わない |
| 書き込み | — | 行わない |
| 削除 | アプリアンインストール時 | OS 管理下で自動削除 |

## 件数上限

該当なし (key-value 1 件のみ)。

## 採用理由

[初期素案][sketch] は永続化対象として `@AppStorage` / `shared_preferences` を明記する一方、UI はシステム自動追従 (手動切替 UI なし) と規定する。両者を整合させる方針として、本仕様は永続化キーと値域のみを定義し、UI 切替手段は提供しない。設定画面は素案の 4 画面構成 + 計算ツールに含まれないため作成しない。
