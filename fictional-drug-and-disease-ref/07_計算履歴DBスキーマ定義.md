# 計算履歴 DB スキーマ定義

親プラン: [05_医薬品・疾患マスタアプリ詳細化プラン.md][plan] (計算ツール画面、永続化設計)

## 目的

BMI / eGFR / CrCl の計算ツールで実行した過去の計算内容 (入力値と結果) を永続化し、タップで入力を復元して再計算できるようにする。患者情報を繰り返し計算する医療従事者のワークフローを想定。

計算式は本仕様の対象外。BMI (体重/身長²)、eGFR (日本人係数式)、CrCl (Cockcroft-Gault 式) を採用する方針が [初期素案][sketch] で確定しており、具体式は実装側 README に明記する。

## テーブル定義

### `calculation_history` (Flutter drift / iOS SwiftData 共通スキーマ)

| カラム | 型 | 制約 | 説明 |
|-------|----|------|------|
| `id` | String | PRIMARY KEY | UUID v4 (クライアント生成) |
| `calcType` | String | NOT NULL, CHECK (`calcType` IN (`bmi`, `egfr`, `crcl`)) | 計算種別 |
| `inputs` | String (JSON) | NOT NULL | 入力値の JSON シリアライズ (`calcType` により構造が異なる) |
| `result` | String (JSON) | NOT NULL | 計算結果の JSON シリアライズ (数値 + 分類など) |
| `calculatedAt` | DateTime | NOT NULL | 計算実行日時 (ISO 8601 UTC) |

### インデックス

- `idx_calc_type_date ON calculation_history(calcType, calculatedAt DESC)` — 計算種別別の時系列表示

### `inputs` / `result` JSON スキーマ (calcType 別)

**BMI**:
- `inputs`: `{ heightCm: Double, weightKg: Double }`
- `result`: `{ bmi: Double, category: String }`
  - `category` 値域: `"underweight" | "normal" | "overweight" | "obese_1" | "obese_2" | "obese_3" | "obese_4"`

**eGFR** (日本人係数式):
- `inputs`: `{ ageYears: Int, sex: String, serumCreatinineMgDl: Double }`
  - `sex` 値域: `"male" | "female"`
- `result`: `{ eGfrMlMin173m2: Double, ckdStage: String }`
  - `ckdStage` 値域: `"G1" | "G2" | "G3a" | "G3b" | "G4" | "G5"`

**CrCl** (Cockcroft-Gault 式):
- `inputs`: `{ ageYears: Int, sex: String, weightKg: Double, serumCreatinineMgDl: Double }`
  - `sex` 値域: `"male" | "female"`
- `result`: `{ crClMlMin: Double }`

## 使用箇所

### 画面配置: 計算ツール画面内の折りたたみ履歴セクション

- 画面: `CalcToolView` (Flutter `/calc` / iOS `CalcToolView`)
- 配置: 各 `calcType` フォームの下に「履歴」折りたたみセクション。デフォルト折りたたみ状態、タップで展開
- 履歴項目表示: `calculatedAt` 時刻 + 入力値サマリ + 結果 (1 行)
- タップ挙動: `inputs` を現在のフォームに復元し、即座に再計算 (`result` は再計算値で上書き、古い履歴は削除しない)
- スワイプ挙動: 単件削除

## ライフサイクル

| 操作 | トリガ | DB 操作 |
|-----|-------|--------|
| 作成 | フォーム入力完了 + 計算実行時 | `INSERT` で新規 UUID + 入力値・結果・現在時刻を格納 |
| 読み取り | 計算ツール画面展開時 | `SELECT * FROM calculation_history WHERE calcType = ? ORDER BY calculatedAt DESC LIMIT 50` |
| 削除 (単件) | 履歴項目スワイプ削除 | `DELETE FROM calculation_history WHERE id = ?` |
| 自動削除 | 件数上限超過時 | 最古の `calculatedAt` から順に削除 (`calcType` ごとに独立) |

読み込み時に毎回 `ORDER BY ... DESC LIMIT` で取得するため、画面表示パフォーマンスは 50 件で一定。

## 件数上限

`calcType` ごとに独立して上限 50 件 (BMI 50 + eGFR 50 + CrCl 50 = 最大 150 行)。FIFO 自動削除。

## 採用理由

タップで入力復元 + 即再計算、スワイプで削除、各 `calcType` 50 件上限を採用。代替案: タップで結果再表示のみ (入力復元なし) は患者パラメータを少し変えて再計算するワークフローが成立しないため却下、タップで削除のみは再利用が目的のため却下。
