## 疾患データモデル

### 分類軸

| 軸 | 値の数 | 画面分岐への影響 | 出典 |
|----|-------|------------------|------|
| ICD-10 章 (icd10Chapter) | 22 (`Icd10Chapter` enum、`CHAPTER_I`〜`CHAPTER_XXII`) | カテゴリフィルター、関連疾患グループ化 | [ICD-10 2013年版 厚労省](https://www.mhlw.go.jp/toukei/sippei/) |
| 診療科 (medicalDepartment) | 16 (`MedicalDepartment` enum、複数値可) | 検索フィルター、関連薬品絞り込み | 医療機関標榜診療科 (一般的区分) |
| 慢性/急性 (chronicity) | 4 (`Chronicity` enum) | 治療経過・予後 UI の強弱 | 一般医学分類 |
| 感染性 (infectious) | 2 (Boolean) | 予防・伝播経路項の要否 | ICD-10 第 I 章 (A00-B99) の扱い |

`icd10Code` (A00-Z99 等の細分コード) は架空データで整合性を保証できないため本モデルでは保持しない。分類は `icd10Chapter` のみで行う。

### 共通フィールド

ICD-10 22 章と一般疾患リファレンスの共通項を抽出。

| # | フィールド | 型 | 必須 | 備考 |
|---|------------|-----|------|------|
| 1 | id | `String` | 必須 | `disease_NNNN` 4 桁ゼロ埋め、0 から `件数 - 1` まで連番 (穴なし) |
| 2 | name | `String` | 必須 | 疾患名 (架空) |
| 3 | nameKana | `String` | 必須 | 検索用 |
| 4 | nameEnglish | `String?` | 任意 | 英語名 |
| 5 | icd10Chapter | `Icd10Chapter` | 必須 | 列挙型 `CHAPTER_I`〜`CHAPTER_XXII`、ワイヤ値 `chapter_i`〜`chapter_xxii` |
| 6 | medicalDepartment | `List<MedicalDepartment>` | 最低 1 要素必須 | 列挙型、複数診療科可 |
| 7 | chronicity | `Chronicity` | 必須 | 列挙型 |
| 8 | infectious | `Boolean` | 必須 | `CHAPTER_I` は `true` 必須 |
| 9 | synonyms | `List<String>` | `=emptyList()` | 別名・俗称 |
| 10 | summary | `String` (MD) | 必須 | 1-2 段落の概説 (自由記述) |
| 11 | epidemiology | `EpidemiologyInfo?` | 任意 | `CHAPTER_I` / `CHAPTER_XV` は非 null 必須 |
| 12 | etiology | `String` (MD) | 必須 | 原因・病態 (自由記述) |
| 13 | symptoms | `SymptomInfo` | 必須 | 下記構造 |
| 14 | diagnosticCriteria | `DiagnosticCriteriaInfo` | 必須 | 下記構造。`required` ≥1 は全件必須 |
| 15 | requiredExams | `List<Exam>` | 最低 1 要素必須 | 全 disease で必須 |
| 16 | severityGrading | `SeverityInfo?` | 任意 | null なら E10 非表示。`CHAPTER_II` / `CHAPTER_IX` は非 null 必須。非 null 時は `grades` ≥2 必須 |
| 17 | differentialDiagnoses | `List<String>` | `=emptyList()` | 鑑別疾患名 |
| 18 | complications | `List<String>` | `=emptyList()` | 合併症名 |
| 19 | treatments | `TreatmentInfo` | 必須 | 下記構造 |
| 20 | prognosis | `String?` (MD) | 任意 | 予後 (自由記述)。`CHAPTER_II` は非 null かつ非空必須 |
| 21 | prevention | `List<String>` | `=emptyList()` | 予防箇条書き (各要素は短文、MD 非対象)。`infectious=true` または `EpidemiologyInfo.riskFactors` 非空時に最低 1 要素必須 |
| 22 | relatedDrugIds | `List<String>` | `=emptyList()` | 医薬品モデルへの参照 (ダングリング禁止)。`CHAPTER_V` は向精神薬を最低 1 件含み、`CHAPTER_XV` は妊婦禁忌薬を含めない |
| 23 | relatedDiseaseIds | `List<String>` | `=emptyList()` | 関連疾患 (`disease_NNNN`、ダングリング禁止) |
| 24 | revisedAt | `String` (YYYY-MM-DD) | 必須 | 日は `01` 固定 |
| 25 | disclaimer | `String` | 必須 (デフォルト `Disclaimer.SHORT`) | root に必ず付与。値: `"FICTIONAL DATA - NOT FOR MEDICAL USE / 架空データ・医療判断には使用不可"` |

### 入れ子構造 (全て `@Serializable data class`)

#### EpidemiologyInfo

```kotlin
@Serializable
data class EpidemiologyInfo(
    val prevalence: Prevalence? = null,              // 有病率 (構造化)
    val onsetAgeRange: OnsetAgeRange? = null,        // 好発年齢。CHAPTER_XV は非 null 必須
    val sexRatio: SexDistribution? = null,           // 男女比。CHAPTER_XV は非 null 必須
    val riskFactors: List<String> = emptyList(),     // 各要素は短文 (MD 非対象)。CHAPTER_I は非空かつ感染経路キーワードを含む
)

@Serializable
data class Prevalence(
    val rate: Double?,                               // 例 12.5
    val denominator: Int?,                           // 例 100000 (10 万対)
    val unit: PrevalenceUnit = PrevalenceUnit.PER_POPULATION,
    val label: String,                               // 表示用 (例: "人口10万対 12.5")
)

@Serializable
enum class PrevalenceUnit {
    /** 人口対 */ @SerialName("per_population") PER_POPULATION,
    /** 患者対 */ @SerialName("per_patient") PER_PATIENT,
    /** 出生対 */ @SerialName("per_birth") PER_BIRTH,
}

@Serializable
data class OnsetAgeRange(
    val minAgeYears: Int?,
    val maxAgeYears: Int?,
    val label: String,    // 表示用 (例: "40-60 代")
)

@Serializable
data class SexDistribution(
    val maleRatio: Int,    // 整数比 (例 "2:1" の 2)
    val femaleRatio: Int,  // 整数比 (例 "2:1" の 1)
    val note: String? = null,    // "男性優位" 等の補足
)
```

#### SymptomInfo

```kotlin
@Serializable
data class SymptomInfo(
    val mainSymptoms: List<String>,                    // 主症状 (各要素は短い症状名、MD 非対象)。最低 1 要素必須、CHAPTER_V は最低 3 要素必須
    val associatedSymptoms: List<String> = emptyList(), // 随伴症状
    val onsetPattern: OnsetPattern? = null,            // 発症様式 (enum)。CHAPTER_I は非 null 必須
)

@Serializable
enum class OnsetPattern {
    /** 急性発症 */ @SerialName("acute") ACUTE,
    /** 亜急性発症 */ @SerialName("subacute") SUBACUTE,
    /** 慢性経過 */ @SerialName("chronic") CHRONIC,
    /** 間欠性 */ @SerialName("intermittent") INTERMITTENT,
    /** 再発性 */ @SerialName("relapsing") RELAPSING,
}
```

#### Exam

```kotlin
@Serializable
data class Exam(
    val name: String,                     // 検査名 (例: HbA1c、心エコー)
    val category: ExamCategory,
    val typicalFinding: String,           // 典型所見 (平文、架空数値)
    val referenceRange: String? = null,   // 基準値 (平文)
)
```

`CHAPTER_IX` の disease は `requiredExams` 内に最低 1 件 `category = ExamCategory.IMAGING` を含む必要がある。

#### SeverityInfo

```kotlin
@Serializable
data class SeverityInfo(
    val gradingSystem: String,    // 例: "NYHA 分類"
    val grades: List<Grade>,      // 最低 2 要素必須 (severityGrading 非 null 時)
)

@Serializable
data class Grade(
    val label: String,                // 例: "I 度"
    val criteria: String,             // 判定基準 (平文)
    val recommendedAction: String,    // 記法: Markdown (1-2 文)
)
```

#### DiagnosticCriteriaInfo

```kotlin
@Serializable
data class DiagnosticCriteriaInfo(
    val required: List<String>,                    // 必須項目 (箇条書き、各要素は短い Markdown)。全 disease で最低 1 要素必須
    val supporting: List<String> = emptyList(),    // 補助項目
    val notes: String? = null,                     // 記法: Markdown (補足説明)
)
```

#### TreatmentInfo

```kotlin
@Serializable
data class TreatmentInfo(
    val pharmacological: List<PharmaTreatment> = emptyList(),     // 薬物療法。CHAPTER_IV は最低 1 要素必須
    val nonPharmacological: List<TreatmentSection> = emptyList(), // 非薬物療法 (手術/生活指導/リハビリ 等)
    val acutePhaseProtocol: List<ProtocolStep> = emptyList(),     // 急性期対応。CHAPTER_IX かつ chronicity = ACUTE は最低 1 要素必須
)

@Serializable
data class PharmaTreatment(
    val drugCategory: String,                 // 例: "経口血糖降下薬"
    val drugIds: List<String> = emptyList(),  // 内部 Drug id 参照 (ダングリング禁止)
    val indication: String,                   // 適応 (平文)
    val notes: String,                        // 記法: Markdown
)

@Serializable
data class TreatmentSection(
    val heading: String,                      // 例: "食事療法", "理学療法", "手術"
    val items: List<String> = emptyList(),    // 箇条書き (各要素は短文、MD 非対象)
    val description: String? = null,          // 記法: Markdown (補足段落)
)

@Serializable
data class ProtocolStep(
    val order: Int,             // 1, 2, 3 ...
    val action: String,         // 記法: Markdown (1-2 文)
    val target: String? = null, // 目標値 (例: "収縮期血圧 140 mmHg 未満")
)
```

### 列挙型定義

全て `@Serializable enum class`、JSON ワイヤ値は `@SerialName` で **英語 snake_case** に固定する。


**sort 基準**: `Icd10Chapter.compareTo()` (= enum.ordinal = 宣言順 = 章番号 1〜22 順) を採用。`@SerialName` 文字列順 (snake_case lexicographic) は採用しない (`chapter_i < chapter_iii < ... < chapter_x < chapter_xv` で章番号順とずれる)。

**codeRange / label メタデータ**: `/v1/categories` 用の `Icd10ChapterEntry(roman, code, label)` に必要な ICD-10 章コードレンジ (`code`、例: `"A00-B99"`) と日本語章名 (`label`、例: `"感染症および寄生虫症"`) は `Icd10Chapter.kt` に extension property として配置する。

### 分類軸別の追加フィールド (必須化条件)

| 分類 | 必須化されるフィールド |
|------|------------------------|
| 感染性疾患 (`icd10Chapter = CHAPTER_I`) | `infectious = true` / `epidemiology` 非 null / `epidemiology.riskFactors` ≥1 かつ感染経路キーワードを含む / `symptoms.onsetPattern` 非 null / `prevention` ≥1 (riskFactors 非空時) |
| 新生物 (`icd10Chapter = CHAPTER_II`) | `severityGrading` 非 null / `prognosis` 非 null かつ非空 |
| 内分泌代謝疾患 (`icd10Chapter = CHAPTER_IV`) | `treatments.pharmacological` 最低 1 要素 |
| 精神・行動障害 (`icd10Chapter = CHAPTER_V`) | `symptoms.mainSymptoms` 最低 3 要素 / `relatedDrugIds` に向精神薬 (`PSYCHOTROPIC_*`) を最低 1 件含む |
| 循環器系 (`icd10Chapter = CHAPTER_IX`) | `severityGrading` 非 null / `requiredExams` 内に `ExamCategory.IMAGING` 最低 1 件 |
| 循環器系 急性 (`icd10Chapter = CHAPTER_IX` かつ `chronicity = ACUTE`) | `treatments.acutePhaseProtocol` 最低 1 要素 |
| 妊娠・分娩・産褥 (`icd10Chapter = CHAPTER_XV`) | `epidemiology` 非 null / `epidemiology.onsetAgeRange` 非 null / `epidemiology.sexRatio` 非 null / `relatedDrugIds` に妊婦禁忌薬 (`precautionsForSpecificPopulations[].category` に `PREGNANT` を含む薬品) を含めない |

### Markdown 記述規約 (Fixture 生成指示)

文体は **常体**、見出しは `##` / `###` まで。

| フィールド | 推奨構造 | 目安行数 | ダミー例 (架空) |
|-----------|---------|---------|---------------|
| summary | 段落 (見出しなし) | 3-6 | `テスタマイン症は、インスリン様ホルモンの作用不全を特徴とする慢性代謝疾患である。成人発症が多く、糖代謝異常に加えて脂質代謝異常を合併する。` |
| etiology | 段落 + 必要に応じ箇条書き | 3-10 | `原因は遺伝的素因と生活習慣の相互作用と考えられている。\n- 第 7 染色体上の架空遺伝子多型\n- 長期の過栄養状態` |
| DiagnosticCriteriaInfo.required[] | 短文または短い Markdown (箇条 1 要素分) | 1 | `空腹時サンプル値**≧ 126 mg/dL**` |
| DiagnosticCriteriaInfo.supporting[] | 同上 | 1 | `HbA1c 相当値**≧ 6.5%**` |
| DiagnosticCriteriaInfo.notes | 段落 | 2-5 | `本基準は架空値であり、実臨床の診断には使用しない。` |
| prognosis | 段落 | 2-5 | `早期発見により予後は良好である。診断 5 年後の寛解率は**約 65%** と報告される (架空)。` |
| Grade.recommendedAction | 1-2 文 | 1-2 | `生活指導のみで経過観察する。3 ヶ月毎の再評価を推奨。` |
| TreatmentSection.description | 段落 | 2-5 | `標準体重 × 25 kcal/kg を目安とし、糖質 50% / 脂質 25% / タンパク質 25% の配分を推奨する。` |
| ProtocolStep.action | 1-2 文 | 1-2 | `初期輸液として生理食塩液**500 mL/h** を開始する。` |
| PharmaTreatment.notes | 段落 1-2 文 | 1-3 | `第一選択薬として本剤を使用する。投与 4 週後に効果判定を行う。` |

平文 (Markdown 非対象) の文字列フィールド: `SymptomInfo.mainSymptoms[]`, `SymptomInfo.associatedSymptoms[]`, `differentialDiagnoses[]`, `complications[]`, `synonyms[]`, `EpidemiologyInfo.riskFactors[]`, `prevention[]`, `TreatmentSection.items[]`, `Exam.typicalFinding`, `Exam.referenceRange`, `Grade.criteria`, `PharmaTreatment.indication`, `ProtocolStep.target`。

モデルのどのブロックを画面のどの位置に出すか、表示条件付きで定義。表示条件は全て「型上の存在性」(`null` / 空配列 / enum 値) で表現。

| # | 表示ブロック | 使用フィールド | 表示条件 |

## 組合せ数計算 (疾患)

### 計算前提

「データ件数」= 1 Disease レコード単位 (対応する医薬品側は [03_医薬品モデル詳細.md `#combination-counts`](03_医薬品モデル詳細.md#combination-counts) 参照)。UI 発火条件 (非空 / 非 null / 特定 enum 値) を全網羅しつつ、分類軸 × UI ブロックの組合せを最小件数で満たすカバレッジを求める。素案は「疾患 30 件」だが、以下の計算で**実現不可能**であることを示す。

### (A') 分類軸の全値カバレッジ

### (D') 重複圧縮後の最小値

単純加算 = A'(22) + B'(28 の残差 約 10) + C'(30) = 約 62。
UI ブロック 8 (治療組合せ 2^3) と 5 (ExamCategory 5 種) は兼用限界がある → 下限は**約 60-65 件**。

**疾患推奨件数: 80 件** (余剰 15 件でバリエーション確保)。
