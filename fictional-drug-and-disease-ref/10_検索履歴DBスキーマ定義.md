# 検索履歴 DB スキーマ定義

親プラン: [05_医薬品・疾患マスタアプリ詳細化プラン.md][plan] (検索画面機能要件、永続化設計)

## 目的

検索画面で実行した過去の検索条件 (キーワード + フィルタ + ソート) を永続化し、ワンタップで再検索できるようにする。医薬品タブと疾患タブでそれぞれ独立した履歴を持つ。

## テーブル定義

### `search_history` (Flutter drift / iOS SwiftData 共通スキーマ)

| カラム | 型 | 制約 | 説明 |
|-------|----|------|------|
| `id` | String | PRIMARY KEY | UUID v4 (クライアント生成) |
| `target` | String | NOT NULL, CHECK (`target` IN (`drug`, `disease`)) | 検索対象タブ |
| `query` | String (JSON) | NOT NULL | 検索条件の JSON シリアライズ (キーワード + フィルタ + ソート) |
| `searchedAt` | DateTime | NOT NULL | 検索実行日時 (ISO 8601 UTC) |
| `totalCount` | Int | NOT NULL | 検索結果件数 (envelope の `total_count` から取得) |

### インデックス

- `idx_search_target_date ON search_history(target, searchedAt DESC)` — タブ別の時系列ソート

### `query` JSON スキーマ

JSON フィールド名は camelCase。値は API クエリパラメータの wire format (snake_case `@SerialName` / enum 名 / `-` プレフィックス sort) と完全一致させ、復元時は **camelCase フィールド名 ↔ snake_case クエリパラメータ名** の名前マップ変換のみで `/v1/drugs` / `/v1/diseases` をリクエスト可能とする。

**target = `drug`** (`/v1/drugs` クエリ仕様に対応):

```
{
  keyword: String?,
  keywordMatch: "partial" | "prefix",                      // 既定 "partial"
  keywordTarget: "generic" | "brand" | "both",             // 既定 "both"
  regulatoryClass: String?,                                // 単一: RegulatoryClass の @SerialName 値 (例: "prescription_required")
  dosageForm: String?,                                     // 単一: DosageForm の @SerialName 値 (例: "tablet")
  route: String?,                                          // 単一: RouteOfAdministration の @SerialName 値 (例: "oral")
  categoryAtc: String?,                                    // 単一: ATC コード先頭 1 文字 (例: "A")
  therapeuticCategory: String?,                            // 単一: TherapeuticCategory の enum 名 (例: "ALIMENTARY_METABOLISM")
  adverseReactionKeyword: String?,
  precautionCategory: List<String>,                        // 複数: PrecautionPopulationCategory の enum 名 (例: "PREGNANT")
  sort: "-revised_at" | "brand_name_kana" | "atc_code" | "therapeutic_category_name"  // 既定 "-revised_at"
}
```

クエリパラメータ対応 (camelCase → snake_case): `keywordMatch → keyword_match` / `keywordTarget → keyword_target` / `regulatoryClass → regulatory_class` / `dosageForm → dosage_form` / `route → route` / `categoryAtc → category_atc` / `therapeuticCategory → therapeutic_category` / `adverseReactionKeyword → adverse_reaction_keyword` / `precautionCategory → precaution_category` / `sort → sort`。

**target = `disease`** (`/v1/diseases` クエリ仕様に対応):

```
{
  keyword: String?,
  keywordMatch: "partial" | "prefix",                      // 既定 "partial"
  keywordTarget: "name" | "name_english" | "synonyms",     // 既定 "name"
  icd10Chapter: String?,                                   // 単一: Icd10Chapter の @SerialName 値 (例: "i")
  department: String?,                                     // 単一: MedicalDepartment の @SerialName 値 (例: "internal_medicine")
  chronicity: String?,                                     // 単一: Chronicity の @SerialName 値 (例: "chronic")
  infectious: Bool?,
  symptomKeyword: String?,
  onsetPattern: List<String>,                              // 複数: OnsetPattern の enum 名 (例: "ACUTE")
  examCategory: List<String>,                              // 複数: ExamCategory の enum 名 (例: "IMAGING")
  hasPharmacologicalTreatment: Bool?,
  hasSeverityGrading: Bool?,
  sort: "-revised_at" | "name_kana" | "icd10_chapter"      // 既定 "-revised_at"
}
```

クエリパラメータ対応 (camelCase → snake_case): `keywordMatch → keyword_match` / `keywordTarget → keyword_target` / `icd10Chapter → icd10_chapter` / `department → department` / `chronicity → chronicity` / `infectious → infectious` / `symptomKeyword → symptom_keyword` / `onsetPattern → onset_pattern` / `examCategory → exam_category` / `hasPharmacologicalTreatment → has_pharmacological_treatment` / `hasSeverityGrading → has_severity_grading` / `sort → sort`。

各 enum の値域定義は共通基盤・医薬品・疾患モデル仕様を参照: [02_薬品・疾患_詳細画面データ構造設計プラン_共通基盤.md][base] / [03_医薬品モデル詳細.md][drug] / [04_疾患モデル詳細.md][disease]。

## 使用箇所

### 画面配置: 検索画面キーワード欄フォーカス時のドロップダウン

- 画面: `SearchView` (Flutter `/search` / iOS `SearchView`)
- 配置: キーワードテキストフィールドにフォーカスが入ると、フィールド直下に履歴ドロップダウン (最大 5 件表示)
- 履歴項目表示: `searchedAt` 相対時刻 + キーワード抜粋 + フィルタ有無バッジ + `totalCount`
- タップ挙動: `query` を現在のタブ + フォームに復元し、即検索実行

## ライフサイクル

| 操作 | トリガ | DB 操作 |
|-----|-------|--------|
| 作成 | 検索実行時 (画面内検索ボタン押下、またはソフトキーボードの送信アクションキー押下 = Flutter `TextField.onSubmitted` / iOS `onSubmit`)。debounce 入力中の Live 検索は発火対象外 | `INSERT` で新規 UUID + target + query + 現在時刻 + totalCount を格納 |
| 重複排除 | 作成前に同一 target + query の既存行検索 | 存在時は旧行を `DELETE` してから `INSERT` (最新時刻で復活) |
| 読み取り | キーワード欄フォーカス時 | `SELECT * FROM search_history WHERE target = ? ORDER BY searchedAt DESC LIMIT 20` |
| 削除 (単件) | ドロップダウン項目スワイプ / ドロップダウン内「x」ボタン | `DELETE FROM search_history WHERE id = ?` |
| 自動削除 | 件数上限超過時 | `target` ごとに最古から削除 (FIFO) |

## 件数上限

`target` ごとに上限 20 件 (医薬品 20 + 疾患 20 = 最大 40 行)。

## 採用理由

- 配置: キーワード欄フォーカス時のドロップダウン (常時表示セクションは画面占有大、別画面は導線遠いため却下)
- 件数: 20 件 (画面スクロール 1-2 回で把握可能)
- 重複排除: 同一条件はマージ (最新時刻に更新)、履歴汚染防止
- タブ分離: `target` カラムで医薬品/疾患別管理 (タブ切替時に混在表示しない)
