# ブックマーク DB スキーマ定義

親プラン: [05_医薬品・疾患マスタアプリ詳細化プラン.md][plan] (ブックマーク一覧画面、永続化設計)

## 目的

ユーザーがブックマークした医薬品・疾患のリスト最小項目を永続化し、オフラインでもリスト表示を可能にする。詳細表示はオンライン API 取得を基本とし、失敗時はスナップショットをフォールバックとする。

## テーブル定義

### `bookmarks` (Flutter drift / iOS SwiftData 共通スキーマ)

| カラム | 型 | 制約 | 説明 |
|-------|----|------|------|
| `id` | String | PRIMARY KEY | ブックマーク対象 ID (`drug_NNNN` または `disease_NNNN`) |
| `snapshot` | String (JSON) | NOT NULL | API リスト最小項目 (`DrugSummary` / `DiseaseSummary`) のシリアライズ |
| `bookmarkedAt` | DateTime | NOT NULL | ブックマーク日時 (ISO 8601 UTC) |

`id` は `drug_NNNN` / `disease_NNNN` 形式で名前空間分離されているため `id` 単独で一意。種別判定は `id` 接頭辞 (`drug_` / `disease_`) で行い、専用カラムは持たない。

### インデックス

- `idx_bookmarks_id_prefix ON bookmarks(substr(id, 1, 5))` — ブックマーク画面 3 タブ (すべて/医薬品/疾患) 切替用 (`drug_` / `disease_` の先頭 5 文字フィルタ)
- `idx_bookmarks_date ON bookmarks(bookmarkedAt DESC)` — 時系列ソート

タブ別フィルタの SQL 例: `SELECT * FROM bookmarks WHERE id LIKE 'drug_%' ORDER BY bookmarkedAt DESC`。

### `snapshot` JSON の内部スキーマ (id 接頭辞別)

各フィールド定義・型は共通基盤 [02_薬品・疾患_詳細画面データ構造設計プラン_共通基盤.md][base] / 医薬品 [03_医薬品モデル詳細.md][drug] / 疾患 [04_疾患モデル詳細.md][disease] の型規約に従う。API レスポンスの snake_case フィールドはクライアント DTO で decode し、`snapshot` JSON は Flutter / iOS のローカル domain summary 名に合わせた camelCase key で保存する。enum 値は API wire value (`@SerialName`) のまま保持し、画面描画時にローカライズマップで表示文言に変換する。

**`drug_*`** (`/v1/drugs` レスポンスの `DrugSummary` 全 10 フィールド):
```
{ id, brandName, genericName, therapeuticCategoryName, regulatoryClass,
  dosageForm, brandNameKana, atcCode, revisedAt, imageUrl }
```

**`disease_*`** (`/v1/diseases` レスポンスの `DiseaseSummary` 全 8 フィールド):
```
{ id, name, icd10Chapter, medicalDepartment, chronicity, infectious,
  nameKana, revisedAt }
```

## 使用箇所

- ブックマーク一覧画面 (3 タブ): `snapshot` を JSON デシリアライズしてリスト項目を描画
- 医薬品詳細画面 / 疾患詳細画面: ブックマークトグルで `INSERT` / `DELETE`、画面表示時に `SELECT COUNT(*) WHERE id = ?` で状態判定
- 詳細画面のオフラインフォールバック: API 失敗時に `snapshot` で最小項目のみ表示 + 警告バナー

## ブックマーク一覧 UI 要件

- データ取得: 一覧描画はローカル `bookmarks` テーブルの `snapshot` のみを使う。API は詳細画面遷移後に詳細画面側で取得する。
- 並び順: `bookmarkedAt DESC` 固定。ソート UI は表示しない。
- タブ: `すべて` / `医薬品` / `疾患`。`id` 接頭辞 (`drug_` / `disease_`) でフィルタする。
- キーワード検索: 保存済み snapshot の表示名のみを対象にする。医薬品は `brandName` / `genericName` / `brandNameKana`、疾患は `name` / `nameKana`。種別ラベル・接頭辞・内部実装メモは検索対象にも UI 表示にも含めない。
- 件数表示: 現在のタブとキーワード検索を反映した表示件数を検索欄付近に表示する。共通 AppBar には件数・読込状態を表示しない。
- 行表示: 検索結果カードと同じ visual contract を使う。医薬品は画像タイルあり、疾患は画像タイルなし。
- 行タップ: `drug_` は医薬品詳細、`disease_` は疾患詳細へ遷移する。
- 削除: ブックマーク一覧の行スワイプ削除、または詳細画面トグル解除で単件削除する。ブックマーク一覧に全削除 FAB は置かない。
- Loading: shimmer skeleton 行のみを表示し、可視の「読み込み中」ラベルは表示しない。
- Empty: ブックマーク 0 件では「ブックマークがありません」+「医薬品・疾患の詳細画面でブックマークを追加すると、ここに一覧表示されます」+「検索画面へ」を表示する。
- Search zero: ブックマーク自体は存在するがキーワード検索結果が 0 件のときは「一致するブックマークがありません」を表示する。
- Error: ローカル DB 読取失敗、または `snapshot` JSON decode 失敗時は「ブックマークを読み込めません」+ 再試行を表示し、失敗をログ出力または上位 Error state に保持する。

## ライフサイクル

| 操作 | トリガ | DB 操作 |
|-----|-------|--------|
| 作成 | 詳細画面のブックマークボタンタップ時 | `INSERT` + `snapshot` に当該 API レスポンスのリスト最小項目を JSON エンコードして格納 |
| 更新 | 詳細画面遷移時の `/v1/drugs/{id}` / `/v1/diseases/{id}` 成功時 | `UPDATE bookmarks SET snapshot = ? WHERE id = ?` (自動 sync) |
| 削除 | ブックマーク画面スワイプ削除 / 詳細画面トグル解除 | `DELETE FROM bookmarks WHERE id = ?` |

## 件数上限

上限なし。架空データ上限は医薬品 120 + 疾患 80 = 計 200 件。全件ブックマークしても `snapshot` 平均 800 バイトで DB サイズ約 160 KB。

## 採用理由

スナップショット方式 (リスト最小項目 JSON 保存) を採用。代替案: フル entity 保存はモデル拡張で線形増加・API 改訂との sync ロジック複雑のため却下、ID のみ保存はオフラインリスト表示不可・タップ毎 API 取得が N レイテンシのため却下。
